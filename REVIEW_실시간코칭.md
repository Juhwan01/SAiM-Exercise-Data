# 실시간 코칭 룰 — 한 줄씩 같이 읽는 회의 문서

## 이 함수는 언제 호출되나
사용자가 한 세트 끝내고 RPE를 입력했을 때 1회 호출. 받아오는 건 지금 운동 상황 한 묶음(운동명, 세트번호, 무게, 1RM 추정, 목표 RPE, 입력 RPE, 통증·수면·근육통, 질환 등). 돌려주는 건 다음 세트 무게, 휴식 초, 폼 큐, 호흡 가이드, 디로드 권고, 차단 신호.

## 흐름
6단계를 순서대로 돈다. 1단계에서 차단 나오면 거기서 즉시 종료(2단계 이후 안 돔).

```
1. 안전 게이트 — 운동해도 되는지 검사
2. 다음 세트 무게
3. 휴식 시간
4. 폼·호흡·체크리스트
5. 디로드 권고
6. 보조(수분)
```

# 1단계 · 안전 게이트

## Q01 — 부상 후 첫 3일은 운동 중단
- 발동: 사용자가 "다친 시점(시간)" 입력
- 동작:
  - 다친 지 < 72시간 → 운동 중단, 보호 단계 안내(보호·거상·압박·교육)
  - 72시간 지남 → 점진 부하 재개 단계 안내
- 예시: "어제 발목 삐었음(24시간 전)" → 즉시 차단, PEACE 단계 가이드 표시 후 종료
- 출처: Dubois 2019 BJSM ★★★
- 결정거리: 사용자가 시간을 정확히 입력 못 할 때 어떻게 받을지(어제/오늘/이번주 같은 버킷?)

## E15 — 통증 패턴으로 부상/근육통 구분
- 발동: 통증 부위·패턴·강도 중 하나라도 입력
- 동작:
  - 무릎·어깨·팔꿈치·손목·허리·고관절 통증 → "부상 의심" → 차단(Q01로 전환)
  - 패턴이 "예리한" 또는 "국소" → 부상 의심 → 차단
  - "양측 대칭 둔통" → 정상 근육통, 가벼운 운동 가능
  - 그 외 → "불확실" → 강도 5↑이면 중단 권고
- 예시: "왼쪽 무릎 예리하게 아픔" → 차단, Q01 안내로 넘김
- 출처: Cheung 2003 ★★☆
- 결정거리: "양측 대칭"인지를 사용자가 어떻게 답하지? 라벨 디자인 필요

## L05 — 당뇨 환자 안전 가드
- 발동: 사용자 질환에 "당뇨" 있음
- 동작:
  - 공복혈당 > 250 + 케톤(+) → 운동 보류, 내분비내과 의뢰
  - 망막증 합병증 → 고강도/Valsalva/점프/머리 숙임 금지, 안과 의뢰
  - 자율신경병증·심한 신경병증·족부궤양 → 체중부하 운동 제한
  - 인슐린·SU 복용 + 운동 전 혈당 < 100 → 탄수 15g 보충 권고
- 예시: 당뇨 + 망막증 → 차단 X, "고강도 금지" 경고 + 안과 의뢰 메시지
- 출처: ACSM 2022 + ADA 2024 + 대한당뇨병학회 2023 ★★★
- 결정거리: 합병증 항목을 온보딩에서 어떻게 수집? 자가 입력 vs 진단명 코드?

## L06 — 고혈압 환자 안전 가드
- 발동: 사용자 질환에 "고혈압" 있음
- 동작:
  - 운동 중 가슴통증·심한두통·시야흐림·호흡곤란·실신감 → 즉시 중단, 119
  - 안정시 BP ≥ 180/110 → 운동 보류, 내과 의뢰
  - 안정시 BP > 160/100 → 심장내과 상담 권고(차단은 X)
  - 항상: Valsalva 호흡 금지 + 저항 운동 RPE 상한을 6으로 강제 하향
  - 베타차단제 복용 → 심박수 대신 RPE로 강도 관리
- 예시: 고혈압 + 목표 RPE 8 입력 → "RPE 6으로 하향" 경고
- 출처: ACSM 2022 + 대한고혈압학회 2022 ★★★
- 결정거리: 안정시 혈압을 매 세션 입력받기 vs 주1회 입력?

## L07 — 이상지질혈증 가드
- 발동: 질환에 "이상지질혈증"
- 동작:
  - LDL > 190 → 운동부하검사 권고
  - 심혈관질환 이력 또는 (당뇨 + 50세↑) → 운동부하검사
  - 항상: 주당 200–300분 유산소, 1,000–2,000 kcal 권고
- 출처: ACSM 2022 + 한국지질학회 2022 ★★★
- 결정거리: LDL 수치를 사용자가 알고 있을까? 모를 때 디폴트 동작?

## L08 — 골관절염(무릎·고관절) 가드
- 발동: 질환에 "골관절염_무릎" 또는 "골관절염_고관절"
- 동작:
  - 야간통·관절잠김·갑작스런 무력감·관절 발적·발열 → 즉시 차단, 정형외과
  - 통증 NRS > 5 → 강도/ROM 감량, RPE 상한 5
  - 깊은 스쿼트·점프·중량 런지 시도 → 회피 경고
  - 무릎 OA → 무릎 굽힘 90° 이내 강제
  - 항상: 통증 NRS 4 이내 유지, 수중운동·태극권 권고
- 예시: 무릎 OA + 깊은 스쿼트 시도 → "이 동작은 회피, 부분 ROM 또는 머신 대체" 경고
- 출처: OARSI 2019 + ACR 2019 ★★★
- 결정거리: 통증 NRS 입력 UI(매 세션? 매 운동?)

## K06 — 수면 부족 시 부상 위험
- 발동: 최근 3일 수면 시간 입력됨
- 동작:
  - 3일 연속 6시간 미만 → 디로드/휴식 권고
  - 최저 5시간 미만 → 강도 RPE 1단계 하향 권고
  - 그 외 → 통과
- 출처: Milewski 2014 ★★★
- 결정거리: 수면 데이터 어디서 받지? 수동 입력 vs 헬스 앱 연동?

## K08 — 근육통(DOMS) 분류
- 발동: 근육통 시작 시점 입력됨
- 동작:
  - 7일 이상 지속 → 부상 의심, 차단 + 의료 평가
  - 24–72시간 → 정상 피크, 가벼운 운동 권고
  - 그 외 → 회복 중
- 출처: Dupuy 2018 ★★☆

# 2단계 · 다음 세트 무게

> 입력 RPE가 없으면 이 단계 통째로 스킵.

## A04 — 사용자가 입력한 RIR을 보정
- 발동: 입력 RPE 있음(RPE를 RIR로 변환 가능한 6.5 이상)
- 동작: 사용자는 보통 RIR을 1회 정도 적게 보고 → 시스템이 보정값을 더해줌
  - 초보 +1.0 / 중급 +0.5 / 고급 +0.25
  - 입력 RIR이 3 이상이면 +0.5 추가
  - 65세 이상이면 +0.5 추가
  - 최대 5까지만 보정
- 예시: 초보가 "RIR 2"라 입력 → 시스템 내부값 RIR 3으로 처리
- 출처: MASS / Bastos 2024 ★★☆
- 결정거리: 보정량(0.25–1.0)이 너무 큰지/작은지, 실제 사용자 데이터로 검증해야 함

## A11 — 단순 식으로 다음 세트 무게 보정
- 발동: 입력 RPE 있음
- 공식: `다음 무게 = 현재 무게 × (1 + (목표 RPE − 입력 RPE) × 0.04)`
  - 0.5점 차이마다 ±2%, 1점 ±4%, 2점 ±8%
- 예시: 100kg × 5렙, 목표 RPE 8 / 입력 RPE 9 → 다음 세트 96kg
- 출처: Helms M&S Pyramid / Ripped Body ★★★

## A12 — 컴파운드는 RTS 룰로 한 번 더 보정
- 발동: 운동 종류 = "compound"
- 동작:
  - 컴파운드 + 8렙 이하 → 1점차당 4.5%
  - 그 외(고렙·아이솔) → 1점차당 2.5%
  - **목표보다 2점 이상 무겁게 나오면 즉시 -8.5% + 세트 종료 권고**
  - A11 결과와 평균내서 다음 무게로 사용
- 예시: 100kg, 목표 RPE 8, 입력 RPE 10 → 즉시 91.5kg + "이번 세트 종료" 권고
- 출처: Tuchscherer RTS ★★☆
- 결정거리: A11(단순)과 A12(RTS) 평균 내는 게 맞나, 아니면 둘 중 하나만 쓸지

## A14 — 고강도 영역에서 한 번 더 깎음
- 발동: 1RM 추정값 있음
- 동작: 무게가 1RM의 85% 이상 또는 5렙 이하면 → A11/A12 결과를 추가로 ×0.95
- 예시: 1RM 100kg, A11 결과 88kg(=88%) → 추가로 83.6kg
- 출처: Tuchscherer 차트 ★★☆
- 결정거리: 1RM을 어떻게 알지(직접 측정 vs 추정 공식)?

## A03 — 톱셋 대비 너무 떨어지면 세트 종료
- 발동: 톱셋 무게·RPE·실제렙 모두 있음
- 동작: 톱셋 추정 1RM 대비 현재 세트 추정 1RM이 92% 미만(=8%↓)이면 세트 종료 권고
- 출처: Tuchscherer / All Things Gym ★★★
- 결정거리: 톱셋 데이터를 사용자가 매번 기억할까? 자동 추적 필요

## (호출 안 됨, 회의에서 결정 필요)

### A01 — RPE 정의 (RPE↔RIR 환산)
- 무엇: RIR = 10 − RPE 환산식
- 왜 안 부름: 코칭 결정에는 안 쓰임. 사용자가 RPE를 그냥 0–10으로 입력
- 어디 쓸 만한가: RPE 입력칸 옆 툴팁/도움말 ("RPE 8 = 한 번 더 가능")
- 결정거리: UI에 노출할지, 어떤 형식으로 띄울지

### A02 — "5렙 RPE 8 목표라면 무게 얼마?" 직접 처방
- 무엇: 1RM × A07(렙·RPE) / 100 으로 다음 세트 무게 계산
- 왜 안 부름: 지금 흐름은 *직전 세트 결과 보고 다음 세트 보정만* 함 (A11/A12). 첫 세트 무게는 사용자 본인이 정한다는 가정
- 결정거리: 첫 세트 무게도 시스템이 직접 처방할지 (그러면 A02·A07 둘 다 진입에 박아야 함)

### A06 — 입력 UI 모드 토글
- 무엇: 사용자 레벨에 따라 RPE 입력 화면을 다르게
  - 초보 → OMNI-RES 시각형 0–10 (그림으로 표현)
  - 중급 → RIR ("남은 가능 횟수 몇 회?")
  - 고급 → RPE 0.5 단위 정밀
- 왜 안 부름: 지금은 사용자가 RPE 0.5 단위로 그냥 입력한다고 가정 (단일 입력 모드)
- 결정거리: 입력 UI 모드 분기를 실제로 만들지

### A07 — RTS 풀 매트릭스 (렙×RPE → %1RM 표)
- 무엇: Tuchscherer 표 그대로 (1렙 RPE10=100% / 5렙 RPE8=81% / 8렙 RPE9=76% 등)
- 왜 안 부름: A02가 참조하는 표인데 A02가 안 불려서 같이 잠자는 셈
- A02 도입 결정 시 자동으로 같이 살아남

### A08 — 정밀 RIR 환산 헬퍼
- 무엇: 0.5 단위 RPE → RIR (RPE 8.5 → RIR 1.5)
- 왜 별도 호출 안 함: A04(RIR 보정) 안에서 이미 부르고 있어서 외부 진입 호출은 필요 없음. 헬퍼 역할
- 결정거리: 없음 (정상)

### A09 — RPE 표 자동 캘리브레이션
- 무엇: "이 사용자는 표대로 나온 RPE 8을 항상 9로 느낀다" 패턴이 3세션 연속이면 표 자체를 -3%p 보정
- 왜 안 부름: **여러 세션 누적 데이터**가 필요. 실시간 코칭은 세트 1회 호출이라 누적 못 함
- 어디서 호출돼야: 별도 백그라운드 작업 또는 세션 종료 후 분석 단계
- 결정거리: 캘리브레이션을 백그라운드로 돌릴지, 사용자 동의 받고 돌릴지

### A13 — 세션 간 RPE drift 보정
- 무엇: 같은 무게×렙에서 세션마다
  - RPE +1 올라감 → 회복 부족 → 다음 세션 무게 -5%
  - RPE -1 떨어짐 → 적응 완료 → +2.5–5%
- 왜 안 부름: 한 세트 안에서 정할 게 아님. **세션 종료 후** 또는 **다음 세션 시작 전**에 평가해야 의미 있음
- 어디서 호출돼야: "다음 세션 시작 처방" 별도 진입 함수
- 결정거리: 세션 시작 처방 진입 함수를 따로 만들지

# 3단계 · 휴식 시간

## C07 — 운동 종류와 강도로 휴식 결정
- 발동: 항상
- 동작:
  - 현재 무게 / 1RM ≥ 85% → "스트렝스 모드" → 240초
  - 컴파운드 + 그 외 → 150초
  - 아이솔레이션 → 90초
- 예시: 80kg 스쿼트 / 1RM 100kg(=80%) → 150초 휴식
- 출처: Schoenfeld 2016 ★★★
- 결정거리: 사용자 목표(다이어트/벌크업/스트렝스)를 직접 받지 않고 강도로만 추정해도 되나?

# 4단계 · 폼·호흡·체크리스트

## E16 — 첫 세트 컴파운드는 셋업 5단계 띄움
- 발동: 세트 번호 = 1 + 컴파운드
- 출력: ①발 위치 → ②그립 너비 → ③척추 중립 → ④광배·코어 잠금 → ⑤호흡 셋팅
- 출처: NSCA / Starting Strength ★★★

## E06–E12 — 종목별 폼 큐
- 발동: 운동명이 매칭되는 종목일 때
  - 스쿼트 → E06 / 벤치 → E07 / 데드 → E08 / OHP → E10 / 풀업·친업 → E11 / 바벨로우 → E12
- 출력: 각 종목 4–5개 외적 큐 리스트 (룰 카드 그대로)
  - 예 스쿼트: "발 한가운데 누르며 일어선다", "가슴을 천장으로"
- 출처: Power Rack Strength / Starting Strength / SBS / PowerliftingTechnique ★★–★★★
- 결정거리: 종목 누락 없는지 (덤벨 운동·머신 운동에 큐 없음)

## E05 — 외적 큐 우선 사용 메타
- 발동: 위 종목 큐가 발동될 때 자동으로 같이 기록
- 출력: "외적 큐 우선" 메타 표시 (별도 메시지 X)
- 출처: Wulf ★★★

## E13 — 호흡 가이드
- 발동: 항상
- 동작:
  - 무게 > 1RM의 80% + 고혈압 없음 → "Valsalva(들숨 후 호흡정지)"
  - 무게 > 80% + 고혈압 있음 → "Valsalva 금지, 들 때 강하게 후 내쉬기"
  - 무게 ≤ 80% → "들 때 호기, 내릴 때 흡기"
- 출처: NSCA / Hackett 2013 ★★★

## E17 — 그립 가이드
- 발동: 운동명이 벤치·데드·풀업·친업
- 출력: 종목별 그립 너비 1줄
- 출처: PowerliftingTechnique / SBS ★★☆

## E03 — 바벨 속도 떨어지면 세트 종료
- 발동: 렙별 바벨 속도 데이터 있음(센서 필요)
- 동작: 첫 렙 대비 마지막 렙 속도 감소율
  - 파워 ≥ 15% / 근력 ≥ 22.5% / 근비대 ≥ 32.5% → 세트 종료
- 출처: Pareja-Blanco 2022 ★★★
- 결정거리: VBT 센서 없는 사용자에겐 영원히 발동 안 됨. 기능 우선순위 결정 필요

## (호출 안 됨, 회의에서 결정 필요)

### E01 — VBT 피로 알람
- 무엇: 렙별 바벨 속도 리스트 받아 첫 렙 대비 마지막 렙 감소율 ≥ 20%면 "피로 감지" 알림
- 왜 안 부름: VBT 센서/카메라 데이터(`렙별_바벨속도_ms`) 없으면 발동 불가. E03가 비슷한 역할(목표별 임계로 세트 종료 권고)을 이미 하고 있어 중복
- 결정거리: VBT 센서 사용자에게만 노출하는 별도 카드로 만들지

### E02 — Load-Velocity Profile (그날의 1RM 추정)
- 무엇: 워밍업 무게/속도 측정 → 선형회귀 → 그날 1RM 추정 (속도 0인 지점의 무게)
- 왜 안 부름: 워밍업 중 여러 무게×속도 샘플 필요. 실시간 코칭은 본 세트 시작 후 호출이라 워밍업 단계가 없음
- 어디서 호출돼야: 세션 시작/워밍업 단계용 별도 진입
- 결정거리: 워밍업 진입 함수가 따로 필요할지

### E04 — 세트 후 시각 피드백
- 무엇: 평균 속도/피크 속도/감소율%를 시각 그래프로 보여주기 (Weakley 2023: 시각 피드백이 동기·수행 향상)
- 왜 안 부름: 결정 로직이 아니라 표시용 데이터 가공. UI 영역
- 결정거리: 세트 결과 화면에 띄울지, 어떤 그래프 디자인으로

### E14 — 가동범위(ROM) 큐
- 무엇: 측정된 ROM 비율(0–1)이 0.85 미만이면 "엉덩이 더 내린다"·"바를 가슴까지" 같은 큐 출력
- 왜 안 부름: ROM 비율을 측정할 카메라/센서 입력이 `CoachingState`에 없음
- 결정거리: 카메라 기반 폼 측정 도입할지, 아니면 사용자 자가 보고로 받을지

# 다른 진입에 박혀야 할 룰 (실시간 코칭 흐름에 안 들어감)

위 단계별 (호출 안 됨) 박스에 푼 A·E 룰 외, 아예 다른 진입(세션 시작 처방·세션 종료 분석·워밍업)이 필요한 룰들:

### C08 — Double Progression
- 무엇: 모든 세트가 렙 상한(예: 8–12렙의 12) 도달 → 다음 세션 무게 +2.5%, 렙 하한(8) 리셋
- 왜 실시간 X: 한 세트 안에서 결정할 게 아님. 세션 종료 후 다음 세션 처방 시점
- 어디서 호출돼야: 세션 시작 처방 진입

### D14 — 첫 세션 무게 클램프
- 무엇: 초보가 무리한 무게 입력 시 종목별 캡(스쿼트 65kg, 데드 70kg 등) × 체중 비례로 떨굼
- 왜 실시간 X: 첫 세션 시작 전 처방 단계
- 어디서: 루틴 추천(이미 5단계에서 사용 중) — 실시간엔 안 들어감

### D15 — 증량 폭
- 무엇: 다음 세션 무게 = 현재 + 5kg(하체) / +2.5kg(상체). 실패 시 절반으로
- 왜 실시간 X: 다음 세션 처방
- 어디서: 세션 시작 처방 진입

### I01 — 초보 Linear Progression
- 무엇: 초보(<6개월) → 자율조절(RPE 보정) 끄고 LP만 사용 (매 세션 고정 +5/+2.5kg 진보)
- 왜 실시간 X: 한 세트 결정이 아니라 프로그램 모델 자체 결정
- 어디서: 루틴 추천(이미 7단계에서 사용 중)

### I04 — ACWR (급성/만성 부하비)
- 무엇: 7일 부하 / (28일 부하 ÷ 4). 0.8–1.3 안전 / 1.5+ 부상 위험
- 왜 실시간 X: 28일 부하 누적 데이터가 필요
- 어디서: 별도 백그라운드 작업 또는 세션 종료 후 분석 진입

### P04 — 관절 모빌리티 처방
- 무엇: 폼 에러(스쿼트 깊이 부족·골반 윙크·등 말림 등) 입력 시 책임 관절(고관절/발목/흉추) 모빌리티 8분 처방
- 왜 실시간 X: 폼 에러 감지가 카메라/사용자 입력 의존. 또 처방은 세션 전 워밍업 시점이 적절
- 어디서: 워밍업 진입, 또는 세션 종료 후 "다음 세션 워밍업 처방"

### P05 — 정적 스트레칭 경고
- 무엇: 사용자가 워밍업에 정적 스트레칭 ≥ 45초 넣으면 "최대 근력 -5.4%" 경고 + 동적 대체 권고
- 왜 실시간 X: 워밍업 시점에 띄울 경고. 본 세트 진입한 후엔 이미 늦음
- 어디서: 워밍업 진입

### I02 — 템포 표기 파싱
- 무엇: "3-1-1-0" 표기를 (이심성·바닥·구심성·탑) 초로 분해
- 왜 안 부름: 사용자가 템포를 입력해야 의미. 템포 입력 UI 자체가 아직 미설계
- 어디서: 세션 시작 처방에서 운동마다 템포 함께 노출할지 결정 필요

### D04 — Flexible DUP
- 무엇: "오늘 컨디션 좋음/보통/피곤" 입력 → 그날 세션 타입 (근력/비대/회복) 동적 분기
- 왜 실시간 X: 컨디션 입력은 세션 시작 시점에 받아야 함
- 어디서: 세션 시작 처방 진입

# 큰 결정거리

1. **세션 시작 처방 진입 함수가 따로 필요** — A13·C08·D15·I01·D04·I02·P04·P05 가 다 거기로 가야 함. 지금은 "1세트 첫 호출"만 있는데, 그 전에 "세션 시작 1회 호출" 진입을 분리해야 함

2. **세션 종료 후 분석 진입도 필요** — A09(RPE 표 캘리브레이션), I04(ACWR), A13(drift 평가)는 누적 데이터 위에서 동작. 백그라운드로 돌릴지 사용자 액션으로 트리거할지

3. **VBT 센서·ROM 측정 없는 사용자 폴백** — E01·E02·E04·E14는 센서/카메라 의존. 센서 없는 사용자는 어떤 대체 흐름?

4. **안전 가드 입력값 수집 UI** — L05–L08가 요구하는 혈당·BP·LDL·NRS·합병증 코드를 온보딩에서 어떻게 받을지, 매 세션 다시 받을지, 주1 갱신할지

5. **첫 세트 무게 직접 처방 (A02 도입 여부)** — 지금은 사용자가 첫 세트 무게를 정함. A02/A07 도입하면 시스템이 1RM·목표 RPE·목표 렙 받아 직접 처방 가능
