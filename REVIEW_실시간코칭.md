# 실시간 코칭 룰베이스 — 한 줄 한 줄 회의용 정리

> 진입 함수: `rule_engine.realtime_coaching(state) -> CoachingResult`
> 룰 수: 50개 (`coaching_rules/` 6개 모듈)
> 이 문서는 각 룰이 **어떤 입력을 보고 / 어떤 분기로 / 어떤 출력을 내는지**만
> 사실 그대로 옮긴 회의 자료다. 옳고 그름 판단은 회의에서.

---

## 0. 전체 흐름 (`realtime_coaching.py`)

입력으로 `CoachingState` (지금 세트 상황) 1개를 받고, 6단계를 순서대로 실행한다.
1단계에서 차단 신호가 나오면 거기서 끝, 나머지는 안 돌린다.

```
[1] 안전 게이트   →  차단되면 즉시 종료
[2] 다음 세트 무게
[3] 휴식 시간
[4] 폼·호흡·체크리스트
[5] 디로드 권고
[6] 보조(수분) 안내
```

같은 룰 ID라도 루틴 추천에서 쓰이는 함수와는 **별도 구현**이다 (서로 호출 안 함).

## 1단계 — 안전 게이트 (`coaching_rules/safety.py` + form_cues.E15)

### Q01 — 급성 부상 처치 (PEACE & LOVE)
- 출처: Dubois 2019 BJSM ★★★
- **조건**: `state.통증_시작_h` 가 입력됨
- **분기**:
  - `< 72h` → 단계="PEACE", 차단=True (운동 중단)
  - `>= 72h` → 단계="LOVE", 차단=False
- **결과**: 차단 시 결과의 `차단됨=True`, `차단_이유=PEACE...`, 4단계 행동 리스트 반환

### E15 — 통증 패턴 분류 (안전 게이트로 사용)
- 출처: Cheung 2003 ★★☆
- **조건**: 통증부위/패턴/강도 중 하나 이상 입력됨
- **분기**:
  - 통증부위 ∈ (무릎·어깨·팔꿈치·손목·허리·고관절) **또는** 패턴 ∈ (예리한·국소) → "부상_의심" → 차단
  - 패턴 = "양측대칭" → "DOMS_정상"
  - 그 외 → "불확실"
- **결과**: 부상_의심이면 즉시 차단 + Q01 PEACE 단계로 전환 권고

### L05 — 제2형 당뇨 가드
- 출처: ACSM 2022 + ADA 2024 + 대한당뇨병학회 2023 ★★★
- **조건**: `state.질환` 안에 "당뇨" 포함
- **분기 (적색기는 차단)**:
  - 공복혈당 > 250 mg/dL **AND** 케톤(+) → 차단, 내분비내과 즉시 상담
  - 합병증 = "증식성_망막증" → 고강도/Valsalva/점프/머리숙임 금지, 안과 의뢰 (차단 X, 주의)
  - 합병증 ∈ (자율신경병증·신경병증_심·족부궤양) → 체중부하 운동 제한, 내분비내과 의뢰
  - 인슐린/SU 복용자 + 운동 전 혈당 < 100 → 탄수 15g 보충 권고
- **결과**: 차단/주의/의뢰 dict

### L06 — 고혈압 가드
- 출처: ACSM 2022 + 대한고혈압학회 2022 ★★★
- **조건**: `state.질환`에 "고혈압" 포함
- **분기**:
  - 급성 증상(가슴통증·심한두통·시야흐림·호흡곤란·실신감) 1개라도 → 차단, 119 의뢰
  - 안정시 SBP ≥ 180 또는 DBP ≥ 110 → 차단, 내과 의뢰
  - SBP > 160 또는 DBP > 100 → 차단 X, 심장내과 상담 권고
  - 기본: Valsalva 회피 경고 + 저항운동 RPE 상한 6 강제
  - 약물별: 베타차단제 → HR 대신 RPE / 이뇨제 → 탈수주의 / 알파차단제 → 쿨다운 5분
- **결과**: `RPE_상한=6.0` 이 결과의 경고에 들어가서 사용자의 목표 RPE를 내림

### L07 — 이상지질혈증 가드
- 출처: ACSM 2022 + 한국지질학회 2022 ★★★
- **조건**: `state.질환`에 "이상지질혈증" 포함
- **분기**:
  - 급성 증상 → 차단, 응급실
  - LDL > 190 → 운동부하검사 권고
  - CVD 이력 또는 (당뇨+50세↑) → 운동부하검사 권고
  - 기본: 주당 200~300분 유산소, 1,000~2,000 kcal 권고
- **결과**: 의뢰/주의 메시지

### L08 — 골관절염(무릎·고관절) 가드
- 출처: OARSI 2019 + ACR 2019 ★★★
- **조건**: `state.질환`에 "골관절염_무릎" 또는 "골관절염_고관절"
- **분기**:
  - 적색기 신호(야간통·관절잠김·갑작스런 무력감·관절 발적·발열·외상성 부종) → 차단, 정형외과
  - 통증 NRS > 5 → 강도/ROM 감량, RPE 상한 5
  - 운동명에 (깊은_스쿼트·점프·착지·중량_런지·갑작스런_방향전환) 포함 → 회피 경고
  - 무릎 OA → 무릎 굴곡 ≤ 90° 강제
  - 기본: RPE 5~7, 통증 NRS ≤ 4, 수중운동/태극권 권고

### K06 — 수면 부족 위험
- 출처: Milewski 2014 ★★★
- **조건**: `state.최근_3일_수면_h` 입력됨
- **분기**:
  - 최근 3일 모두 < 6h → 위험=True, 디로드/휴식 권고
  - 최저값 < 5h → 위험=True, 강도 RPE 1단계 하향 권고
  - 그 외 → 위험=False
- **결과**: 차단은 하지 않음, 제안에 추가만

### K08 — DOMS 분류
- 출처: Dupuy 2018 ★★☆
- **조건**: `state.근육통_시작_h` 입력됨
- **분기**:
  - ≥ 7일 (168h) → "부상_의심", 차단, 의료 평가 의뢰
  - 24~72h → "DOMS_피크", 가벼운 운동 권고
  - 그 외 → "DOMS_회복중"

## 2단계 — 다음 세트 무게 (`coaching_rules/rpe.py`)

> **전제**: `state.입력RPE` 가 있어야 함. 없으면 이 단계 통째로 스킵.

### A04 → A11 → (A12) → A14 → A03 순서로 한 무게에 누적 적용

#### A04 — RIR 과소예측 보정
- 출처: MASS / Bastos 2024 ★★☆
- **조건**: 입력 RPE를 RIR로 변환한 값(`a08_precise_rir`)이 None 아닐 때
- **분기 (보정값 base)**:
  - 레벨="초보" → +1.0
  - 중급 → +0.5
  - 고급 → +0.25
  - reported_rir ≥ 3 → +0.5 추가
  - 65세 이상 → +0.5 추가
- **결과**: `corrected_rir = min(reported + base, 5.0)`. 이 값으로 내부 RPE 재계산.

#### A11 — Helms 단순 식
- 출처: Helms M&S Pyramid / Ripped Body ★★★
- **조건**: 입력 RPE 있음 (항상 적용)
- **공식**: `next_kg = current_kg × (1 + (목표RPE − 입력RPE) × 0.04)`
  - 0.5점 차이 = ±2% / 1점 = ±4% / 2점 = ±8%
- **결과**: 다음 세트 추천 무게 (kg, 소수 1자리)

#### A12 — RTS 보정 (컴파운드일 때만 추가 발동)
- 출처: Tuchscherer RTS ★★☆
- **조건**: `state.운동종류 == "compound"`
- **분기**:
  - 컴파운드 + 렙 ≤ 8 → 1 RPE 차당 4.5%
  - 그 외(아이솔/9렙↑) → 1 RPE 차당 2.5%
  - **delta ≤ -2.0** (목표보다 2점 이상 무거움) → 즉시 -8.5% + 세트 종료 권고
- **결과**: A11과 RTS 결과를 평균내서 다음 무게로 사용

#### A14 — 고강도 추가 보정
- 출처: Tuchscherer 차트 ★★☆
- **조건**: `state.최대무게_추정_kg > 0`
- **분기**:
  - 무게/1RM ≥ 0.85 **또는** 렙 ≤ 5 → 추가로 ×0.95
  - 그 외 → 변경 없음
- **결과**: A11/A12로 나온 무게를 한 번 더 내림

#### A03 — 누적 피로 종료 트리거
- 출처: Tuchscherer / All Things Gym ★★★
- **조건**: 톱셋 무게/RPE/실제렙 모두 있음
- **공식**:
  - 톱셋 e1RM = 톱셋무게 × (1 + 톱셋RIR/30)
  - 현재 e1RM = 현재무게 × (1 + 현재RIR/30)
  - 현재/톱셋 < 0.92 (= 8% 떨어짐) → True
- **결과**: True면 `세트_종료_권고 = True`

> **참고**: A01·A02·A06·A07·A08·A09·A13 은 모듈 안에 함수로 존재하지만 진입 함수에서 직접 호출하지 않음 (A04 안에서 A08 만 호출됨). UI 옵션·표 룩업·다른 룰의 보조용.

## 4단계 — 폼·호흡·체크리스트 (`coaching_rules/form_cues.py`)

### E16 — 컴파운드 셋업 5단계
- 출처: NSCA / Starting Strength ★★★
- **조건**: `state.세트번호 == 1` AND `운동종류 == "compound"`
- **결과**: ["①발 위치", "②그립 너비", "③척추 중립", "④광배/코어 잠금", "⑤호흡 셋팅"]

### E06~E12 — 종목별 폼 큐
- 출처: Power Rack Strength / Starting Strength / SBS / PowerliftingTechnique 외 ★★~★★★
- **조건**: `state.운동명` 매칭
  - 스쿼트 → E06
  - 벤치프레스 → E07
  - 데드리프트 → E08
  - 오버헤드프레스/OHP → E10
  - 풀업/친업 → E11
  - 바벨로우 → E12
- **결과**: 각 종목 4~5개 외적 큐 리스트 (룰 카드 한 줄 설명 그대로)
  - 예: 스쿼트 → ["발 한가운데를 발로 누르며 일어선다", "가슴을 천장으로 보낸다는 느낌"]

### E05 — 외적 큐 우선 (메타)
- 출처: Wulf ★★★
- **조건**: 위 종목 큐 발동되면 자동
- **결과**: `외적 큐 우선 사용` 한 번 기록 (메타 표시)

### E13 — Valsalva 호흡
- 출처: NSCA / Hackett 2013 ★★★
- **조건**: 항상 발동
- **분기 (무게/1RM)**:
  - > 80% AND 고혈압 없음 → "들숨 후 호흡정지(Valsalva) → 하강 → 상승 후 배출"
  - > 80% AND 고혈압 있음 → "Valsalva 금지. 들 때 강하게 후 내쉬기"
  - ≤ 80% → "들 때 호기, 내릴 때 흡기"
- **결과**: 호흡 가이드 문자열 1줄

### E17 — 그립 가이드
- 출처: PowerliftingTechnique / SBS ★★☆
- **조건**: `state.운동명 ∈ (벤치프레스·데드리프트·풀업·친업)`
- **결과**: 종목별 그립 너비 권장 1줄

### E03 — VBT 속도 손실 종료 트리거
- 출처: Pareja-Blanco 2022 ★★★
- **조건**: `state.렙별_바벨속도_ms` 리스트가 있음
- **공식**: `(첫렙속도 − 마지막렙속도) / 첫렙속도`
- **분기 (목표="근력" 기준 22.5%)**:
  - 파워 ≥ 15% / 근력 ≥ 22.5% / 근비대 ≥ 32.5% / 건강증진 ≥ 20% → 세트 종료 권고
- **결과**: True면 `세트_종료_권고 = True`

> E01·E02·E04·E14 는 함수 정의는 있지만 진입에서 직접 호출 X (VBT 가공·외부 데이터 의존).

## 5단계 — 디로드 권고 (`coaching_rules/deload.py` + misc.B04)

### G02 — 기본 트리거
- 출처: Israetel / Bell 2024 ★★★
- **조건**: 항상 평가
- **분기 (둘 중 하나라도)**:
  - `state.최근_RPE_drift ≥ 2.0`
  - `state.관절통_NPRS ≥ 4`
- **결과**: True면 제안에 "이번 주말 디로드 권장" 추가

### G04 — 주간 부하 +10% 검사
- 출처: Gabbett ACWR ★★★
- **조건**: 지난주 볼륨 > 0
- **공식**: `(이번주 − 지난주) / 지난주`
- **분기**: > 0.10 → 경고 메시지 "주간 볼륨 +X% — 10% 상한 초과"

### G07 — 다지표 디로드 (Delphi)
- 출처: Bell 2023 Delphi Consensus ★★★
- **조건**: 항상 평가, 5개 지표 카운트
  - `RPE_drift ≥ 1.5`
  - 최근 3일 중 < 6h 수면 일수 ≥ 4일 (※ 코드는 3일치 입력에서 카운트)
  - `관절통_NPRS ≥ 4`
  - `동기 ≤ 2/5`
  - `최근_PR_경과일 ≥ 14`
- **분기**: 히트 ≥ 2개 → 디로드 권고

### B04 — MRV 초과
- 출처: Israetel ★★★
- **조건**: `state.이번주_볼륨_세트` 채워짐
- **임계 (근육군당 세트/주)**: 가슴 22 / 등 25 / 어깨 22 / 이두 26 / 삼두 24 / 다리·둔근·햄 20 / 복근 25 / 기타 18
- **분기**: 어떤 근육군이라도 임계 도달 시 → 디로드 권고

## 진입에서 호출되지 않는 룰 (정의만 있음, 회의 시 검토 필요)

`coaching_rules/` 안에 함수가 있지만 `realtime_coaching` 흐름에 안 박혀 있는 룰들:

| 룰 ID | 위치 | 용도 (모듈 주석) |
|---|---|---|
| A01 | rpe.py | RPE→RIR 정의 (UI 라벨용) |
| A02 | rpe.py | RPE×렙 무게 추천 (`a02_target_weight_kg`) |
| A06 | rpe.py | 입력 UI 모드 토글 (OMNI/RIR/정밀) |
| A07 | rpe.py | RTS 매트릭스 (A04 보정과 별도) |
| A08 | rpe.py | 정밀 RIR 환산 (A04 안에서만 호출됨) |
| A09 | rpe.py | RPE 표 캘리브레이션 |
| A13 | rpe.py | 세션 간 drift → 다음 세션 무게 (실시간보다 후속 세션용) |
| C08·D14·D15 | progression.py | 다음 세션 증량 결정 (코칭이 아니라 다음 세션 시작 시점) |
