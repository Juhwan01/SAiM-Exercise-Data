# 운동 루틴 추천 룰 — 한 줄씩 같이 읽는 회의 문서

## 이 함수는 언제 호출되나
사용자가 온보딩에서 프로필을 완성했을 때 1회. 받아오는 건 사용자 한 사람 정보(성별·체중·키·나이·훈련 개월·목표·주가용일·세션 가용분·체지방률·1RM·부상이력·질환·혈압·LDL·장비). 돌려주는 건 주간 프로그램 한 묶음(BMR·TDEE·목표 칼로리·분할 형태·추천 프로그램·주간 스케줄·시작 무게·근육군별 주간 세트·디로드 주기·운동 풀·워밍업·카디오·회복 가이드·교육 메시지).

## 흐름
13단계를 순서대로. 1단계에서 차단 나오면 처방 정지하고 의료 의뢰만 띄움.

```
1.  안전 게이트 (50세↑·청소년·당뇨·고혈압·이상지질혈증·골관절염)
2.  레벨 분류 (초보·중급·고급)
3.  영양 (BMR → TDEE → 목표 칼로리)
4.  분할/프로그램 선택
5.  시작 무게 + 증량 폭
6.  강도/볼륨
7.  주기화/디로드
8.  운동 선택
9.  워밍업/스트레칭
10. 카디오
11. 회복/수면
12. 부상 이력별 가산
13. 행동·교육 메시지
```

| 건강증진 | 변동 없음 | 12–20렙 / 50–67% / 8–12세트 | 컴 150·아이 90 | 폴백 D06 | 주 2회 30분 |

**목표 무관 (어느 목표여도 동일하게 처방되는 단계)**:
- 1단계 안전 게이트 — 질환·나이만 봄

# 1단계 · 안전 게이트

## L02 — 50세 이상은 보수적 처방
- 발동: 나이 50 이상
- 동작: 주 빈도 2–3, 렙 8–12, 세트 2–3, %1RM 60–80, 단백질 1.2–1.6 g/kg
  + "첫 8주 적응기" 경고 추가
- 출처: ACSM 2026 / ESPEN ★★★
- 결정거리: 50세 컷오프가 맞나(60·65세 다른 디테일 가이드도 있음)

## L03 — 18세 미만은 1RM 테스트 잠금
- 발동: 나이 18 미만
- 동작: 1RM 측정 불가, RPE 기반 강도, 렙 6–15, "성장판 안전·폼 우선·코치 감독 필수" 경고
- 출처: NSCA Youth Position Statement ★★★
- 결정거리: 미성년자 가입 정책 자체가 있는가? 약관과 충돌하지 않는지

## L05 — 당뇨 환자 처방
- 발동: 질환에 "당뇨"
- 동작: 유산소 3–7회/주(연속 비활동 ≤ 2일), 저항 2–3회, 저항 RPE 상한 7, 유산소 75–150분/주, 8–10종목 1–3세트, "식후 30분 가벼운 걷기" 권고
- 회피 조건 메시지: 망막증·자율신경병증·공복혈당>250+케톤
- 출처: ACSM 2022 + ADA 2024 + 대한당뇨병학회 2023 ★★★

## L06 — 고혈압 환자 처방
- 발동: 질환에 "고혈압"
- 동작: 유산소 5–7회, 저항 2–3회, 저항 RPE 상한 6, %1RM 60–70, 렙 8–12, Valsalva 금지, 유산소 90–150분/주
- **즉시 차단 조건**: 안정시 SBP ≥ 180 또는 DBP ≥ 110 → 처방 중단, 내과 의뢰
- 출처: ACSM 2022 + 대한고혈압학회 2022 ★★★
- 결정거리: 안정시 혈압을 온보딩에서 입력받을 건지(미입력자는 차단 조건 미발동)

## L07 — 이상지질혈증 처방
- 발동: 질환에 "이상지질혈증"
- 동작: 유산소 5–7회, 저항 2–3회, 유연성 2–3회, RPE 상한 6, 유산소 200–300분/주(최소 150), 주당 1,000–2,000 kcal 소비
- 추가: LDL > 190이면 의료 의뢰 메시지 추가
- 출처: ACSM 2022 + 한국지질학회 2022 ★★★

## L08 — 골관절염(무릎·고관절) 처방
- 발동: 질환에 "골관절염_무릎" 또는 "골관절염_고관절"
- 동작: 저항 2–3회, 유산소 3–5회, 유연성 매일, RPE 5–7, 통증 NRS 4 이내, %1RM 60–70, 렙 8–15
- 무릎 OA → 무릎 굽힘 90° 이내
- 회피 동작: 깊은 스쿼트·점프·착지·중량 런지·갑작스런 방향전환
- 권장: 수중운동·태극권
- 출처: OARSI 2019 + ACR 2019 ★★★

# 3단계 · 영양 (BMR → TDEE → 목표 칼로리)

## K19 vs K20 — BMR 공식 자동 분기
- 발동: 항상
- 동작:
  - 체지방률 입력 안 함 → K19 (Mifflin-St Jeor)
    - 남: `10×체중 + 6.25×키 − 5×나이 + 5`
    - 여: 위 식 −161
  - 체지방률 입력함 → K20 (Katch-McArdle)
    - `370 + 21.6 × (체중 × (1 − 체지방률/100))`
- 예시: 남 70kg, 175cm, 30세, 체지방률 미입력 → K19 → 1,649 kcal
- 출처: Mifflin 1990 ★★★ / Aragon ISSN 2017 ★★★

## K22 — 활동 카테고리 자동 분류
- 발동: 직업 + 주당 운동 빈도 입력
- 동작:
  - 육체노동 + 주3–5회↑ 운동 → 매우 활발
  - 주6–7회 운동 → 활발
  - 주3–5회 → 보통
  - 주1–3회 → 가벼움
  - 그 외 → 좌식
- 출처: FAO/WHO/UNU 2001 ★★☆

## K21 — TDEE 계산
- 동작: TDEE = BMR × PAL
  - 좌식 1.2 / 가벼움 1.375 / 보통 1.55 / 활발 1.725 / 매우 활발 1.9
- 예시: BMR 1,649 + "가벼움" → 1,649 × 1.375 ≈ 2,267 kcal
- 출처: FAO/WHO/UNU 2001 ★★★

## 목표 칼로리 가산 (룰 외, 진입 함수에 박힌 로직)
- 다이어트 −500 / 벌크업 +300 / 근육량 증가 +200 / 그 외 0
- 결정거리: 가산 폭이 적정한지(다이어트 -500이 너무 빡센 경우 있음)

# 4단계 · 분할/프로그램 선택

## D05 — 분할 형태
- 동작:
  - 가용일 ≤ 3 또는 초보 → "풀바디 주N"
  - 가용일 4 → "상하분할 주4"
  - 가용일 5↑ → "PPL 주N"
- 출처: Schoenfeld 2016 / Israetel ★★★

## 프로그램 선택 분기 (D06/D07/D08/D10 + 폴백)
- 초보 → D06 Starting Strength
- 스트렝스 + 중급 → D07 5/3/1 Wendler
- 스트렝스 + 그 외 → D08 GZCLP
- 벌크업/근육량 증가 + 중급 + 가용일 ≥ 4 → D10 PHUL/PHAT
- 그 외 모두 → D06 (폴백)

## D06 — Starting Strength
- 출력: A루틴(스쿼트·벤치·데드) / B루틴(스쿼트·OHP·파워클린 또는 바벨로우), 주 3회, 세션당 +2.5–5kg, 6–12개월 운영
- 출처: Rippetoe ★★★

## D07 — 5/3/1 Wendler
- 출력: 트레이닝 맥스 = 1RM × 0.9
  - 1주: 5렙×3 (65/75/85% TM)
  - 2주: 3렙×3 (70/80/90)
  - 3주: 5/3/1+ AMRAP (75/85/95)
  - 4주: 디로드 (40/50/60)
- 출처: Wendler ★★★

## D08 — GZCLP
- 출력: T1(메인 5+렙 탑셋) / T2(보조 10+렙) / T3(아이솔 15+렙), 주 4회
- 출처: r/Fitness wiki ★★☆

## D10 — PHUL/PHAT
- 동작: 가용일 ≤ 4 → PHUL / 5↑ → PHAT, 근력일+근비대일 결합
- 출처: Norton / Campbell ★★☆

## (호출 안 됨)
- D01 Linear Periodization, D03 Block, D09 nSuns, D04 Flexible DUP — 코드는 있는데 분기에서 안 뽑힘

# 5단계 · 시작 무게

## 분기 1: 중급 이상 + 1RM 입력함
- 시작 무게 = 1RM × 0.8 (운동마다)
- 결정거리: 0.8 고정이 맞나? 종목별로 다른가?

## D11 — 남성 초보 시작 무게
- 출력: 스쿼트 20·벤치 20·OHP 20·데드 40 (kg, 빈 바벨 기준)
- 출처: StrongLifts 5x5 ★★☆

## D12 — 여성 초보 시작 무게
- 동작: 헬스장에 15kg 여성용 바 있으면 바=15kg, 없으면 20kg, 데드 35kg
- 출처: IPF Technical Rules 2024 ★★★
- 결정거리: 사용자가 "헬스장 여성 바 유무"를 어떻게 알지? 디폴트는 어느 쪽?

## D14 — 첫 세션 무게 상한 클램프
- 동작: 종목별 캡(스쿼트 65/35, 데드 70/40, 벤치 40/25, OHP 30/18) × (체중/기준체중)
  - 기준체중 남 70kg, 여 55kg
- 사용자가 더 높게 입력했어도 캡까지 떨굼 + 메모
- 예시: 남 80kg가 첫 스쿼트 100kg 요청 → 클램프 후 약 74kg, "첫 세션 상한" 메모
- 출처: Rippetoe ★★☆

## D15 — 증량 폭 (초기)
- 동작:
  - 하체(스쿼트·데드): 초기 +10kg, 이후 +5kg
  - 상체(벤치·OHP·로우): 초기 +5kg, 이후 +2.5kg
- 진입에선 "초기 폭"만 사용. 이후 폭은 미구현
- 출처: Rippetoe ★★☆

# 7단계 · 주기화/디로드

## 레벨별 주기화 모델
- 초보 → I01 (Linear Progression)
- 중급 → D02 (DUP)
- 고급 → D03 (Block)

## D02 — DUP (중급)
- 출력: 월 비대(8–12렙, 67–75%) / 수 근력(3–6렙, 85–92%) / 금 파워(1–3렙, 90–100%)

- 무엇: 렙 하한 2세션 연속 미달 → -7.5%
- 왜 안 부름: 매 세션 후 처방 단계 룰

> **공통**: C08·C12·C13·C14·C15는 모두 "다음 세션 처방" 진입에 묶여야 함. 현재 그 진입 자체가 없음

## 주기화 모델 (D01·D03·D04·D09·D13)

### D01 — Linear Periodization (12주 사이클)
- 무엇: 1–4주 비대(8–12렙·67–75%) → 5–8주 근력(3–6렙·85–92%) → 9–12주 파워(1–3렙·90–100%)
- 왜 안 부름: 진입 7단계는 레벨로만 모델 분기 (초보 LP / 중급 DUP / 고급 Block). 사용자가 "12주 LP 사이클" 선택할 옵션 없음
- 결정거리: D01을 어느 사용자에게 띄울지

### D03 — Block Periodization (이미 호출됨)
- 진입 7단계에서 고급 사용자에게 호출 중. 정정: 이건 호출됨

### D04 — Flexible DUP
- 무엇: 사용자가 "오늘 컨디션 좋음/보통/피곤" 입력 → 그날 세션 타입 (근력/비대/회복) 동적 분기
- 왜 안 부름: 컨디션 입력은 매 세션 시작 시점에 받아야. 추천은 1회 호출이라 매일 묻지 못 함
- 어디서 호출돼야: 매 세션 시작 진입

### D09 — nSuns LP
- 무엇: 5/3/1 정체기 돌파용. 주 6일·메인+보조 모두 5/3/1 패턴
- 왜 안 부름: 진입 4단계 분기에 D09가 없음. 5/3/1(D07) → GZCLP(D08) → PHUL/PHAT(D10) 만 있음
- 결정거리: D07 stall 시 자동으로 D09로 전환할지

### D13 — 빈 바벨 워크업
- 무엇: 빈 바 → 50% → 70% → 90% → 100% 워크업 시퀀스
- 왜 안 부름: 추천 9단계는 I06(컴파운드 워크업)을 쓰는데, I06이 더 자세한 워크업(빈봉×5×2 + 40/60/80%)이라 D13 중복
- 결정거리: D13은 삭제해도 되는 중복 룰일 수 있음

## 운동 선택 보조 (F05·J04·J07·E14·E17)

### F05 — SFR (자극/피로 비율)
- 무엇: 운동별 SFR 점수 (1–5). 컴파운드 SFR 보통, 머신/아이솔 SFR 우수
- 왜 안 부름: 진입 8단계에서 운동 풀 선택 시 SFR 가중치를 안 씀. J05(컴파운드:아이솔 비율)만 사용
- 결정거리: 부상·피로 누적 사용자에게 SFR 높은 운동 우선 추천할지

### J04 — 편측 vs 양측 운동
- 무엇: 좌우 불균형 ≥ 10% → 편측 비중 ↑ (양측 50:편측 50). 그 외 75:25
- 왜 안 부름: `좌우_불균형_pct` 입력 자체가 진입에서 안 쓰임
- 결정거리: 좌우 불균형을 측정해서 받을지(앱에서 어떻게?)

### J07 — 근육군 매핑 (운동 → primary/secondary)
- 무엇: 백스쿼트=다리·둔근(primary), 복근·햄(secondary). 가중치 1.0/0.5
- 왜 안 부름: 진입 6단계 시작 볼륨(B02)을 근육군 단위로만 처방. 운동→근육 매핑은 안 씀
- 어디서 호출돼야: 누적 볼륨 추적 시 (운동 1세트 → 근육군별 가중치 분배)

### E14 — 가동범위(ROM) 디폴트
- 무엇: "풀ROM 우선, 신장강조 부분반복은 옵션" 메시지
- 왜 안 부름: 진입에서 ROM 안내 문구를 안 띄움
- 결정거리: 사용자에게 ROM 가이드 메시지 노출할지

### E17 — 그립 너비 가이드
- 무엇: 운동별 권장 그립 (벤치 어깨너비 1.5배 / 데드 어깨너비 후크 / 풀업 광배 와이드 / 풀업 이두 내로우)
- 왜 안 부름: 진입에서 그립 안내를 안 띄움 (실시간 코칭에선 이미 사용 중)
- 결정거리: 추천에서도 운동마다 그립 1줄 띄울지

## 디로드 트리거 다양화 (G03·G04·G06·G08·G09·G11)

### G03 — 디로드 처방 방법 (볼륨 -50%, 강도 -10–20%)
- 무엇: G05보다 더 일반적인 처방
- 왜 안 부름: 진입은 G05(세트 ½, 무게 ×0.95, 7일)를 더 구체적으로 사용. G03 중복
- 결정거리: G05 제거하고 G03만 쓸지, 또는 그 반대

### G04 — 주간 부하 +10% 체크
- 무엇: 이번주 / 지난주 > 10% → 부상 위험 경고
- 왜 안 부름: 추천 1회 호출이라 주간 비교 못 함
- 어디서 호출돼야: 주말 자동 분석 진입

### G06 — 고피로 시 강도까지 동시 감축
- 무엇: RPE 평균 > 9 + 수면 < 6h + 관절통 → "고피로 모드" → 세트 ½ + 무게 ½
- 왜 안 부름: 디로드 모드 분기가 진입에 없음. 항상 G05(일반 모드)만
- 결정거리: 사용자 컨디션 입력으로 디로드 모드 동적 분기할지

### G08 — MRV 도달 시 강제 디로드
- 무엇: 주간 세트 ≥ MRV + RPE drift ≥ 1 → 강제 디로드. 또는 6주 초과 시 강제
- 왜 안 부름: 누적 데이터 추적이 진입에 없음
- 어디서 호출돼야: 매 주말 자동 분석 진입

### G09 — 활성 디로드 vs 완전 휴식
- 무엇: 부상·번아웃·수면<5h 지속 → 패시브 1주 / 그 외 → 활성 디로드
- 왜 안 부름: 사용자에게 디로드 모드 묻지 않음
- 결정거리: 디로드 직전에 사용자에게 모드 선택 묻기 vs 자동 분기

### G11 — 미달 누적 디로드 트리거
- 무엇: 다관절 메인리프트 ≥ 2개 stall + 평균 RPE 9–10 → 디로드
- 왜 안 부름: stall 추적이 진입에 없음
- 어디서: 매 세션 후 분석

## 부상 복귀 후속 흐름 (Q02·Q03)

### Q02 — 부상 복귀 단계 전환
- 무엇: 5단계(휴식 → ROM → 무통 저강도 → 점진 부하 → 복귀). 통증 0/10 확인 후 다음 단계
- 왜 안 부름: 진입은 Q04/Q05/Q06으로 단계만 결정하고, 단계 간 전환 로직(Q02)은 안 부름
- 어디서 호출돼야: 부상 사용자의 매 세션 후 "다음 단계 진행 가능?" 체크 진입

### Q03 — 운동 복귀 기준 (4가지 충족)
- 무엇: ROM 정상 + 좌우 근력차 < 10% + 무통 컴파운드 가능 + 2주 무통 → 복귀 허용
- 왜 안 부름: 복귀 판정 자체를 추천이 안 함
- 결정거리: 부상 사용자가 "복귀 준비됐어요" 입력 시 Q03 4가지 체크리스트 띄울지

## 워밍업 다양화 (I07·I08·P02·E02·E09)

### I07 — 아이솔/머신 워밍업
- 무엇: 50% × 8렙 + 80% × 3렙 (1–2 워크업 세트면 충분)
- 왜 안 부름: 진입은 I06(컴파운드 워크업)만 사용. 아이솔 운동에 워크업 처방 없음
- 결정거리: 모든 운동에 워크업 줄지, 컴파운드만 줄지

### I08 — 시간 부족 시 무거운 워크업
- 무엇: 80% × 4렙 1세트만 → 본 세트
- 왜 안 부름: 시간 부족 분기가 진입에 없음
- 결정거리: 세션 가용시간이 짧으면 I08로 자동 전환할지

### P02 — PNF 스트레칭
- 무엇: Contract-relax-contract 기법, ROM 향상에 정/동적보다 우월
- 왜 안 부름: 진입은 정적/동적/SMR/모빌리티만 안내. PNF는 파트너 필요해서 디폴트에서 빠짐
- 결정거리: PNF를 옵션으로 노출할지

### E02 — Load-Velocity 워밍업 캘리브레이션
- 무엇: 워밍업 무게×속도 측정 → 그날 1RM 추정 → 본 세트 무게 ±5% 자동 보정
- 왜 안 부름: VBT 센서 필요
- 결정거리: VBT 센서 사용자 옵션으로

### E09 — RAMP 워밍업
- 무엇: R(심박↑)-A(타깃 활성)-M(가동성)-P(점증세트) 4단계 더 정밀한 워밍업
- 왜 안 부름: I05(일반)+I06(컴파운드 워크업)+P03(SMR)+P04(모빌리티)로 사실상 RAMP를 분리해서 이미 처방. RAMP는 통합 진입이 필요한데 진입 함수가 안 묶음
- 결정거리: I05+I06+P03+P04 통합해서 RAMP로 묶을지, 분리 유지할지

## RPE 표 처방 (rpe_meta.py: A02·A05·A07·A10·A13)

> **큰 결정거리**: **추천에서 RPE 표 기반 구체 무게 처방을 하나도 안 함**. 시작 무게는 D11/D12(빈 바벨) 또는 1RM × 0.8만. 사용자가 "5렙 RPE 8 목표면 80kg" 같은 구체 처방을 못 받음

### A02 — RPE × 렙 → 무게 직접 처방
- 무엇: 1RM × A07(렙·RPE) / 100
- 왜 안 부름: 추천은 렙범위·%1RM 범위만 출력 (예: "6–12렙, 67–85%"). 구체 무게는 사용자에게 떠넘김
- 결정거리: 종목별·세트별 구체 무게까지 처방할지

### A05 — 자율조절 vs 고정 % 모드 선택
- 무엇: 스트렝스 → 자율조절(RPE 보고 매 세션 무게 조정) / 근비대 → 볼륨 우선 (고정 %)
- 왜 안 부름: 진입에서 모드 선택 안 함
- 결정거리: 사용자에게 모드 노출할지, 자동 결정할지

### A07 — RTS 매트릭스 (렙×RPE → %1RM 표)
- 무엇: Tuchscherer 표 (1렙 RPE10=100% / 5렙 RPE8=81% / 8렙 RPE9=76% 등)
- 왜 안 부름: A02 도입 시 같이 살아남

### A10 — 종목·근육군별 변환 보정
- 무엇: 표 결과 % × 종목 보정 계수 (스쿼트 1.00 / 데드 1.01 / 단관절 0.95). 8렙 초과·단관절은 -3%p 추가
- 왜 안 부름: A02·A07 안 부르면 보정할 대상이 없음
- A02 도입 시 같이 활성화

### A13 — 세션 간 RPE drift 보정
- 무엇: 같은 무게×렙에서 세션마다 RPE +1 → -5% / -1 → +2.5–5%
- 왜 안 부름: 누적 데이터 + 세션 시작 시 호출 필요
- 어디서: 매 세션 시작 처방 진입
