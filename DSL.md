# 룰 표기법 (DSL) 정의

이 문서는 운동 코칭 앱이 쓰는 **룰 한 장 짜리 표기법**을 정의한다. 룰 카드 한 개가 한 가지 판단(예: "다음 세트 무게 얼마")을 책임진다. 모든 필드는 **마스터 CSV(`workout_knowledge_base.csv`)의 14개 컬럼** + **`RULEBASE.md` 매트릭스**에서 직접 가져온 것만 쓴다. 새로 만든 정보는 없다.

---

## 1. 룰 카드 한 장의 모양

룰 1개 = `rules/{ID}.yaml` 파일 1개. 153개 룰 = 153개 파일.

```yaml
ID: A11
분류: RPE 자율조절
이름: 세트 간 무게 보정 (Helms 룰)
한줄설명: 목표 RPE 대비 0.5점 차이당 무게 ±2% (1점=±4%, 2점=±8%). 동일 세션의 다음 세트에 즉시 적용

언제_발동: 세트 직후

쓰임새:
  운동_추천: false
  실시간_코칭: true
  지식베이스: true

필요한_정보:
  - 목표 RPE
  - 입력 RPE
  - 현재 무게

하는_일: 다음 세트 추천 무게 (kg)

활용_예시: 5렙 RPE8 목표 + 입력 RPE9 → 다음 세트 -4% (100kg → 96kg). RPE7 입력 → +4% (104kg)

근거:
  출처: Helms (Muscle and Strength Pyramid) / Ripped Body
  주소: https://rippedbody.com/rpe/
  신뢰도: ★★★

적용_위치:
  매트릭스:
    - 목표: 벌크업
      모듈: 세트 후 학습
    - 목표: 스트렝스
      모듈: 세트 후 학습
    - 목표: 근육량 증가
      모듈: 세트 후 학습
  별도_레이어: []
```

---

## 2. 필드 14개 — 어디서 왔는지

### 본문 7필드 (마스터 CSV 그대로)

| 필드 | 마스터 CSV 컬럼 | 무엇 |
|---|---|---|
| `ID` | ID | 룰 식별자. A01~R10 형식 |
| `분류` | 분류 | 큰 카테고리 (예: RPE 자율조절, 영양, 부상 관리) |
| `이름` | 항목 | 룰의 짧은 이름 |
| `한줄설명` | 핵심 룰 (한 줄) | 룰이 무엇을 하는지 한 문장 |
| `언제_발동` | 적용 시점 | 언제 이 룰이 작동하는지 (예: 세션 전, 세트 직후, 주간 단위) |
| `필요한_정보` | 입력값 | 룰이 작동하려면 무엇이 있어야 하는지 |
| `하는_일` | 출력값/동작 | 룰이 결과로 뭘 내놓는지 |
| `활용_예시` | 활용 예시 | 실제 어떻게 쓰이는지 한 가지 사례 |

### 쓰임새 3필드 (마스터 CSV의 Y/-)

| 필드 | 마스터 CSV 컬럼 | 의미 |
|---|---|---|
| `쓰임새.운동_추천` | 운동 추천 | true/false. "오늘 이 무게 이 횟수" 추천에 쓰이면 true |
| `쓰임새.실시간_코칭` | 실시간 코칭 | true/false. 운동 중 즉시 피드백에 쓰이면 true |
| `쓰임새.지식베이스` | 지식베이스 | true/false. "왜?"에 답하는 콘텐츠로 쓰이면 true |

원본 CSV에서 `Y`이면 true, `-`이면 false. 그 외 변형 없음.

### 근거 3필드 (마스터 CSV 그대로)

| 필드 | 마스터 CSV 컬럼 | 의미 |
|---|---|---|
| `근거.출처` | 출처 | 논문·교과서·코치명 |
| `근거.주소` | URL | 원문 링크 |
| `근거.신뢰도` | 신뢰도 | ★★★(메타분석/포지션 스테이트먼트) / ★★☆(좋은 1차 연구·교과서) / ★☆☆(블로그·수정의견) |

### 적용_위치 1필드 (RULEBASE.md 매트릭스에서)

| 필드 | 가져온 곳 | 의미 |
|---|---|---|
| `적용_위치.매트릭스` | RULEBASE.md `## 2. 5목표 × 7모듈 매트릭스` | 이 룰이 어느 (목표, 모듈) 칸에 들어가는지의 목록 |
| `적용_위치.별도_레이어` | RULEBASE.md `## 4. 별도 레이어` | 모집단 분기 / 부상 트리거 / 메타 룰 셋 중 어디 속하는지 |

---

## 3. 5목표 × 7모듈 매트릭스 — 룰이 들어갈 수 있는 칸들

`RULEBASE.md`에 정의된 그대로:

**5목표**: 다이어트, 벌크업, 스트렝스, 근육량 증가, 건강증진
**7모듈**: 온보딩, 세션 시작, 세트별 처방, 세트 후 학습, 메조사이클 관리, 영양·회복, 행동·안전

룰 하나는 여러 칸에 동시에 들어갈 수 있다 (예: A11은 벌크업·스트렝스·근육량 증가의 "세트 후 학습"에 모두 들어감).

---

## 4. 별도 레이어 — 매트릭스 위에 얹는 안전 필터

`RULEBASE.md` 4장에 정의된 3가지:

| 레이어 | 들어가는 룰 | 무엇 |
|---|---|---|
| 모집단 분기 | L02, L03, L04, L05, L06, L07, L08 | 노인·청소년·만성질환 4종(당뇨·고혈압·이상지질혈증·골관절염) 사용자에 대한 안전 필터 |
| 부상 이벤트 트리거 | Q01, Q02, Q03, Q04, Q05, Q06 | "운동 중 통증" 같은 입력에 발동하는 별도 모듈 |
| 메타 룰 | F05 | 운동 선택 단계의 가중치(SFR, 자극 대비 피로 비율) |

> L04(초보/중급/고급 정의)는 매트릭스의 [1] 온보딩에 등장하면서 모집단 분기 별도 레이어에도 동시 등록된다. 두 위치에 모두 속하는 유일한 룰.
>
> J01(부상 부위별 대체 운동표)은 분류상 부상 트리거 후보지만 위 명시 목록에는 포함되지 않아 매트릭스에도 별도 레이어에도 자동 매핑되지 않는다. 자료 보강 결정은 `GAPS.md` §2 참고.

## 5. 신뢰도 표기

| 표기 | 의미 | 예 |
|---|---|---|
| ★★★ | 메타분석, 포지션 스테이트먼트, 합의 가이드라인 | ACSM 가이드라인, Schoenfeld 메타분석 |
| ★★☆ | 좋은 1차 연구, 표준 교과서, 평판 있는 코칭 매뉴얼 | Starting Strength, Helms Pyramid |
| ★☆☆ | 블로그·수정의견·업계 통계 | StrongLifts, IHRSA 보고 |

## 6. 룰 카드 1장 예시 (위 A11과 다른 사례)

```yaml
ID: K19
분류: 영양
이름: Mifflin-St Jeor BMR 공식 (1순위)
한줄설명: 남성은 10×체중+6.25×키-5×나이+5, 여성은 마지막을 -161. 비만/정상 모두 ±10% 오차로 가장 정확

언제_발동: 온보딩 / 체중 변동 시

쓰임새:
  운동_추천: true
  실시간_코칭: false
  지식베이스: true

필요한_정보:
  - 성별
  - 체중(kg)
  - 키(cm)
  - 나이(년)

하는_일: BMR (kcal/일) — 활동계수 곱하기 전 기초대사량

활용_예시: 온보딩에서 키/몸무게/나이/성별 입력 → BMR 자동 산출. 체중 ±2kg 변동 시 재계산

근거:
  출처: Mifflin & St Jeor 1990 (Am J Clin Nutr 51:241-7)
  주소: https://pubmed.ncbi.nlm.nih.gov/2305711/
  신뢰도: ★★★

적용_위치:
  매트릭스:
    - 목표: 다이어트
      모듈: 온보딩
    - 목표: 다이어트
      모듈: 영양·회복
    - 목표: 벌크업
      모듈: 온보딩
    - 목표: 근육량 증가
      모듈: 온보딩
    - 목표: 근육량 증가
      모듈: 영양·회복
    - 목표: 건강증진
      모듈: 온보딩
  별도_레이어: []
```

## 7. 이 표기법의 약속

1. **마스터 CSV가 진실의 원천**. 룰 카드 본문 7필드는 CSV 행과 1:1 대응한다.
2. **매트릭스 정보는 RULEBASE.md에서 가져온다**. 자의로 추가/삭제하지 않는다.
3. **추측 금지**. CSV에 비어 있는 값이면 카드에서도 비어 있다(또는 `null`).
4. **신뢰도는 보존**. 별 1개~3개 그대로.
5. **언어는 평이하게**. 전문용어가 나오면 한 번 풀어 쓴다 (예: RPE → "체감 난이도").

---

## 8. 룰 카드 사용 흐름 (앱이 이 표기법을 어떻게 쓰는가)

```
사용자가 뭘 함  →  앱이 (목표, 모듈) 칸 식별
              →  그 칸의 룰 카드 모음을 가져옴
              →  각 룰의 [필요한_정보] 충족 여부 확인
              →  충족되는 룰의 [하는_일] 실행
              →  결과를 사용자에게 표시 (근거 링크와 함께)
```

별도 레이어(모집단·부상·메타)는 위 흐름에 항상 먼저 적용된다.
